# Fix Hard Command

Tackle complex bugs that require systematic investigation and analysis.

## Usage

```
/fix:hard [description of the issue]
```

## Examples

- `/fix:hard Users getting logged out randomly`
- `/fix:hard Database connection pooling exhausted`
- `/fix:hard Memory leak in background worker`

## Process

1. Reproduce the issue
2. Gather relevant logs and data
3. Analyze stack traces
4. Identify root cause
5. Implement fix
6. Add tests to prevent regression
7. Document the issue and fix

## Output

- Root cause analysis
- Fixed code
- Test cases
- Documentation updates
