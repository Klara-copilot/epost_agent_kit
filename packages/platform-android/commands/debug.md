# Android Debug

Debug Android-specific issues including crashes, Jetpack Compose rendering, Hilt dependency injection, Room database, and performance problems.

## Process

1. **Analyze** — Read error logs, stack traces, or issue description
2. **Locate** — Find relevant Kotlin files, Compose components, or Gradle config
3. **Diagnose** — Identify root cause using systematic debugging methodology
4. **Fix** — Apply targeted fix following Android best practices
5. **Verify** — Run tests to confirm fix

## Focus Areas

- Compose rendering and recomposition issues
- Hilt/DI injection failures
- Room database migration problems
- Retrofit/networking errors
- Gradle build failures
- ProGuard/R8 obfuscation issues
- Memory leaks and performance
