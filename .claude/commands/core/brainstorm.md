# Brainstorm Command

Evaluate technical approaches and design decisions.

## Usage

```
/brainstorm [design question]
/brainstorm how should we implement [feature]?
/brainstorm pros/cons of [approach] vs [alternative]
```

## Your Process

1. Parse the technical question or decision
2. Research relevant patterns and approaches
3. Analyze each option for:
   - Architecture fit with existing system
   - Performance characteristics
   - Maintenance complexity
   - Team expertise match
   - Scalability implications

4. Evaluate trade-offs:
   - Development time
   - Technical debt
   - Future flexibility
   - Risk assessment

5. Provide recommendations with reasoning

## Output

- Multiple viable approaches with pros/cons
- Recommended approach with justification
- Alternative considerations
- Potential gotchas and risks
- Suggested validation steps
- Next steps to proceed

## When to Use

- Choosing between technologies/libraries
- Deciding on architecture patterns
- Evaluating design alternatives
- Resolving technical debates
- Pre-implementation validation
