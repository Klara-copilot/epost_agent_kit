# Accessibility Review

Multi-platform WCAG 2.1 AA compliance review.

<scope>$ARGUMENTS</scope>

**IMPORTANT:** Analyze the skills catalog and activate the skills that are needed for the task.

## Platform Detection

Detect platform from argument or file context:

| Signal | Platform |
|--------|----------|
| `ios` argument, `.swift` files | iOS |
| `android` argument, `.kt` files | Android |
| `web` argument, `.tsx`/`.jsx` files | Web |
| `all` argument | All platforms |

If no platform specified, infer from changed files or ask.

## Process

1. **Detect platform** — from argument or file context
2. **Scan scope** — identify components/views to review
3. **Audit** — check WCAG 2.1 AA compliance per platform:
   - **iOS**: VoiceOver labels, traits, grouping, dynamic type, contrast
   - **Android**: ContentDescription, roles, focus order, touch targets
   - **Web**: ARIA attributes, semantic HTML, keyboard nav, color contrast
4. **Report** — unified findings with severity and fix suggestions
5. **Fix** — offer to auto-fix findings (if applicable)

## Quality Criteria

- Touch targets >= 44x44pt (iOS/Android) / 44x44px (Web)
- Color contrast >= 4.5:1 (text), 3:1 (large text/UI)
- All interactive elements have accessible names
- Focus order matches visual order
- No information conveyed by color alone

## Rules

- Never skip findings — report all issues regardless of severity
- Platform-specific guidance takes priority over generic rules
- Always suggest concrete fixes, not just problem descriptions
