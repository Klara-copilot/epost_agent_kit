# Broad Skill Discovery Across Agent Ecosystem

## Overview

Create a shared `skill-discovery` skill containing platform detection + skill-index lookup + skill loading protocol. Add it to generalist agents so they gain platform awareness on demand — without duplicating the protocol in each agent prompt.

## Current State

- 8 generalist agents have zero platform skills in their `skills:` list
- 4 specialist agents hardcode platform skills (sufficient for them)
- `epost-architect` has a copy-pasted Skill Discovery Protocol (Phase 1 done)
- `.claude/skills/skill-index.json` (47 skills) exists and is filterable by `platforms`, `name`, `keywords`, `agent-affinity`
- `/plan` commands have platform detection (Phase 1 done)

## Target State

- One `skill-discovery` skill — single source of truth for discovery protocol
- 5 high/medium-value generalist agents reference it via `skills:` list
- `epost-architect` inlined protocol replaced with skill reference
- Platform-specific plans, debugging, testing, reviews load correct conventions automatically

- [x] Cross-platform (skill infrastructure)

## Key Dependencies

- `.claude/skills/skill-index.json` must stay current (generated by `generate-skill-index.cjs`)
- Agent `skills:` list is the activation mechanism — no new infrastructure needed

## Success Criteria

- [ ] `skill-discovery` skill exists at `packages/core/skills/skill-discovery/SKILL.md`
- [ ] 5 agents (implementer, debugger, tester, reviewer, orchestrator) have `skill-discovery` in skills list
- [ ] Architect's inlined protocol replaced with skill reference
- [ ] Skill-index.json updated with skill-discovery entry
- [ ] No duplication of platform detection logic across agents

## Risk Assessment

Token overhead: Loading discovery skill + reading 3-5 platform SKILL.md files adds ~3-5K tokens. Acceptable — agents already load 5-7 skills each. Mitigated by "skip if no platform detected" rule.
